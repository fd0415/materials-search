离线阶段 预处理 · 跑一次

把剧集字幕加工成一个「能按语义搜」的片段库。导入素材时跑一遍，新增素材再重跑。

在线阶段 runAgent · 每轮对话

「Agent 味」在于大脑 planTurn 自己判断这轮该重搜 / 换一批 / 只改配文，而非写死流程。红色两步是 LLM 调用。

flowchart TD
    U["用户说一句：message + sessionId"] --> S["getSession 取会话记忆
候选池 pool · 已发 delivered · 当前图 current · 语气 tone · history"]
    S --> P["① 大脑 planTurn · DeepSeek

    P --> D{"action 三选一"}
    D -->|"research 新处境"| R3["把 search 用 bge-m3 算成查询向量

    R2 --> C
    R1 --> C

看懂三个动作就懂了整个 Agent：research=换个处境重新语义检索；next=同一处境再换几张、复用候选池不重搜；recaption=图不动只把配文改狠/改软。
实际 vs spec：spec 里的「离线 LLM 打标签、function-calling 工具循环、ffmpeg 出图」在 v1 尚未落地；当前检索是本地向量余弦召回，规划/配文是让大模型返回 JSON。
